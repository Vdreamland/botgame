# Ruins & Alert Gauge Pre-S1
Special map regions containing relics or packs. Use explore (1 EP, cooldown) to charge the ruin gauge. Free rooms: 15 ruins — 13 relic + 2 pack. Paid rooms: 2 ruins — 1 relic + 1 pack.

| Room | Ruins | Relic ruins | Pack ruins | Guardians | 
| free | 15    | 13          | 2          | 15        | 
| paid | 2     | 1           | 1          | 2         | 

- Gauge max: 3. Each explore: +1 base + explore-affix bonus from equipped relics (minimum 1 per explore).
- Occupant limit: 1 agent at a time. If the occupant leaves or dies, occupant clears and gauge resets to 0.
- Gauge reaches 3 → content acquired automatically, ruin becomes empty.
- Content type (relic/pack) is visible to all; details are masked until game end.

Alert Gauge

Every agent has an alert gauge (0–10). At 10, alert activates and guardians target you.

| Trigger                                         | Change
| Each explore action                             | +2
| Ruin fully cleared (same turn as final explore) | +4 extra
| Turn end while alert is active                  | -4

*Example: clearing 1 ruin (3 explores) = +2, +2, +(2+4) = 10 → instant alert. Gauge decays -4/turn while active; reaches 0 → guardians stop targeting

- WebSocket Events

Event : ruin_state_changed
Key fields : ruinId, gauge, maxGauge, occupiedBy, isEmpty, contentType
When : Explore / cleared / occupant reset

Event : alert_gauge_changed
Key fields : agentId, alertGauge, alertActive
When : Any alert gauge change

- Relics Pre-S1

Stat-boosting gems acquired from relic ruins. Each relic has 0–3 random affixes. Color determines which loadout slot it can be equipped in — not its stats.

Base types — 21 total (7 per color)

| Color     | Theme          | Slot
| R (Red)   | Fire/Attack    | slot 0
| G (Green) | Nature/Balance | slot 1
| B (Blue)  | Wisdom/Defense | slot 2

Affixes — 16 types

Stat : atk
Display name : Ferocious  | Range +1 to +10
Display name : Feeble     | Range -10 to -1

Stat : def
Display name : Unyielding | Range +1 to +5
Display name : Fragile    | Range -5 to -1

Stat : explore
Display name : Insightful | Range +1 (fixed)
Display name : Deluded    | Range -1 (fixed)

Stat : item_atk
Display name : Sharp      | Range +5 to +15
Display name : Blunt      | Range -15 to -5

Stat : max_hp
Display name : Sturdy     | Range +1 to +10
Display name : Sickly     | Range -10 to -1

Stat : max_ep
Display name : Bountiful  | Range +1 to +2
Display name : Withered   | Range -2 to -1

Stat : ep_regen
Display name : Energized  | Range +1 (fixed)
Display name : Sluggish   | Range -1 (fixed)

Stat : hp_regen
Display name : Regenerating | Range +1 to +3
Display name : Festering    | Range -3 to -1

Each relic: 0–3 affixes, randomly drawn from the pool above. Same stat can appear multiple times (values stack).
- Display name format: Ferocious Sturdy Ruby (affixes + gem name). 0-affix relic = gem name only.
- On death: relic drops to its origin ruin (not the death location). The ruin refills and can be explored again.
- Relic details are masked during gameplay. Full details (gem name, affixes, values) revealed at game end via game_settled.

- WebSocket Events
| Event           | Key fields                                | When
| relic_acquired  | agentId, ruinId, instanceId, kind="relic" | Ruin cleared → relic picked up (details masked)
| relic_dropped   | agentId, ruinId, instanceId               | Agent died → relic returned to origin ruin
| relic_discarded | agentId, instanceId, reason="inv_full"    | In-game inventory (5 cap) exceeded

# Loadout

Main Pack 100% + Sub Pack ×0.5 + Relic ×3 ·shared

A pack in the Main slot applies its full effect (100%); a pack in the Sub slot applies an attenuated effect (default ×0.5, varies per pack). The 3 relic slots are shared by Main and Sub.
You cannot run the same category pack in both Main and Sub, and a Main-only pack cannot go in the Sub slot.

# Tiers & Sub slots

Tiers (T1–T3) — T1 is the strongest and T3 the weakest.
Sub attenuation — each pack's description (DB original) also lists its Sub-slot value as Sub …. The attenuation mode varies per pack:

| Mode      | Effect
| ×0.5      | Final value halved (default)
| Partial   | Only certain coefficients reduced
| Sub value | Replaced with a Sub-only value
| Main only | Cannot be placed in the Sub slot
